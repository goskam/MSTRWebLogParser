Security
Everything is client-side javascript. All processing is done in the browser, you can technically open the page in your browser and then use it while offline, or download a local copy of the repository for offline access. No network requests are made by the parser. The upload process parses and loads log files into your browser's memory.

General Usage

Parsing
Upload multiple log file via drag-and-drop into the main upload view. You can also click to select a file to upload, or paste log messages directly into the field below the upload-box.
Searching
Click the search button to open the search view. Searches are real-time, across all log files currently in the dataset (not just the current page). Clear any searches by clearing the search box.
Sorting
Click on a column to change the sort order. The default sort order is using the LID (messages read from the log file in reverse order, with newest on top).
Marking Rows
Mark important messages with the flag button next to a row. This highlights the row for later reference. In future you'll be able to filter/show only flagged rows, as well as export them from the parser .
Removing Rows
Click the trash can to remove rows from the dataset, if those rows are deemed irrelevant. Rows are directly removed from the dataset. The only way to undo this is by re-uploading the log file. Original files are never edited.
Reset
Reset the parser by refreshing the page or clicking the reset button.

Detailed Viewer

Inspecting
Double-click on any row or click the eye button next to a row to open the inspection view for individual messages.
Quick Navigation
Navigate between inspected log messages using the left and right arrow keys. Exit the view using the escape key.
